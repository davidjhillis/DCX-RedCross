Ingeniux CMS Authoring Getting Started

Authoring Basics

Creating Pages with Visual Editing

By default, new web pages can be created anywhere in the Site Tree. However, it is a good idea to keep the site's information architecture in mind when you place a new page.

Important

Do not include the following characters in folder, page, component, or asset names: commas (,), forward slashes (/), backslashes (\), question marks (?), exclamation marks (!), at symbols (@), pound signs (#), dollar signs ($), percent signs (%), carets (^), ampersands (&), asterisks (*), semicolons (;), single quotation marks ('), and double quotation marks (").

Overview

By placing pages logically and consistently, you'll help site visitors find the content they are looking for. The Ingeniux CMS provides multiple ways to create new pages in your Site Tree.

Creating Pages from the Site Toolbar

The Site Toolbar provides quick access to page creation functionality. This method is useful when you want to create a new page as a sibling or child of the currently selected page in the Site Tree.

  1. Navigate to the Site view in the CMS
  2. Select the parent page in the Site Tree
  3. Click the "New Page" button in the toolbar
  4. Choose the appropriate page schema from the list
  5. Enter the page name and complete any required fields
  6. Click "Create" to add the new page to the Site Tree

For automated page creation via API, you can use the following code example:

// Create a new page using the CMS API
const newPage = await cmsAPI.createPage({
  name: 'getting-started',
  schema: 'ArticlePage',
  parentId: 'x123',
  title: 'Getting Started Guide'
});

Creating Pages from Site Tree

You can also create pages directly from the Site Tree context menu. This approach provides more control over where the new page is placed in the site hierarchy.

  1. Right-click on the desired parent page in the Site Tree
  2. Select "New Page" from the context menu
  3. Choose whether to create a child or sibling page
  4. Select the page schema that matches your content type
  5. Complete the required information
  6. Save to add the page to your site

Creating Pages from Content Creation Wizard

The Content Creation Wizard provides a guided approach to creating new pages, particularly useful for users who are new to the CMS or when creating complex page types.

  1. Open the Content Creation Wizard from the navigation pane
  2. Follow the step-by-step prompts to select page location
  3. Choose the appropriate page schema
  4. Fill in the guided form with your content
  5. Review and confirm your selections
  6. Submit to create the new page

Best Practices

Following these best practices will help ensure a well-organized and maintainable site structure:

  • Plan your site's information architecture before creating pages
  • Use descriptive page names that clearly indicate content
  • Follow naming conventions established for your organization
  • Choose the most appropriate page schema for your content type
  • Consider SEO implications when naming pages and organizing structure

Character Restrictions

As noted in the important callout above, certain special characters should never be used in content item names. This helps ensure compatibility across different systems and prevents potential issues during publishing.

Even though Ingeniux CMS 10.3.x and below allow users to save these characters in content item names, they are not to be included. In subsequent releases, users will be unable to save these characters in content item names.

Information Architecture

The way you organize your pages affects not only how easily visitors can find content, but also how efficiently your team can manage the site. Consider creating a logical hierarchy that mirrors how users think about your content.

Keep your site's navigation structure in mind when placing new pages. A well-planned information architecture makes it easier for site visitors to find what they're looking for and improves overall user experience.

Schema Selection

Choosing the right page schema is crucial. Each schema is designed for specific types of content and includes appropriate fields and elements. Consult with your CMS administrator if you're unsure which schema to use for your content.

Here's an example of how to query available schemas programmatically:

// Get available schemas for a parent page
const schemas = await cmsAPI.getAvailableSchemas({
  parentId: 'x123',
  filterBy: 'pageCreationRules'
});

// Filter schemas by type
const articleSchemas = schemas.filter(s => 
  s.type === 'article' || s.type === 'documentation'
);

Next Steps

After creating a new page, you'll want to complete several additional steps to prepare it for publication:

Check Out and Edit

Check out the page to begin editing content. This locks the page to prevent conflicts with other users making simultaneous changes. Add your content using the available page elements defined in your selected schema.

Example of checking out a page programmatically:

// Check out a page for editing
const checkout = await cmsAPI.checkoutPage({
  pageId: 'x456',
  userId: 'user123'
});

if (checkout.success) {
  // Update page content
  await cmsAPI.updatePage({
    pageId: 'x456',
    title: 'Updated Title',
    markForPublish: true
  });
}

Review and Publish

Preview your changes before publishing to ensure everything looks correct. Mark the page for publish when ready, and submit through workflow if configured for your site.
